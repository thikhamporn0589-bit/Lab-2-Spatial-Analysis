### 1. ถ้าเลือกดาวเทียมอีกดวง ผลการวิเคราะห์จะต่างออกไปอย่างไร?

หากเลือกใช้ดาวเทียมอย่าง Sentinel-2 ซึ่งมีความละเอียดเชิงพื้นที่สูงกว่า (10 เมตร) ผลการวิเคราะห์จะสามารถแสดงรายละเอียดของพื้นที่ได้ชัดเจนมากขึ้น โดยเฉพาะในพื้นที่ขนาดเล็ก เช่น แปลงเกษตรหรือพื้นที่เมือง อย่างไรก็ตาม ข้อมูลจะมีขนาดใหญ่ขึ้นและใช้เวลาในการประมวลผลมากขึ้น  

ในทางกลับกัน หากใช้ Landsat 9 ซึ่งมีคุณสมบัติใกล้เคียงกับ Landsat 8 ผลลัพธ์อาจไม่แตกต่างมากนัก แต่จำนวนข้อมูลที่มีอาจน้อยกว่าในบางช่วงเวลา  

ดังนั้น การเลือกดาวเทียมขึ้นอยู่กับความสมดุลระหว่าง “ความละเอียดของข้อมูล” และ “ความสะดวกในการประมวลผล”

### 2. Index ที่เลือกมีข้อจำกัดอะไร? มี Index อื่นที่เหมาะกว่าสำหรับพื้นที่นี้หรือไม่?

NDVI มีข้อจำกัดในการแยกความแตกต่างระหว่างพืชบางประเภท และอาจได้รับผลกระทบจากพื้นผิวดิน (soil background)  

NDWI ที่ใช้ในงานนี้เน้นการตรวจจับน้ำผิวดิน จึงอาจไม่สามารถสะท้อนความชื้นในพืชได้โดยตรง ซึ่งอธิบายได้จากผล correlation ที่เป็นลบ  

NDMI แม้จะสะท้อนความชื้นในพืชได้ดีขึ้น แต่ยังมีข้อจำกัดในพื้นที่ที่มีความหลากหลายของ land cover  

Index อื่นที่อาจเหมาะสม ได้แก่  
- **EVI (Enhanced Vegetation Index)** ซึ่งลดผลกระทบจากดินและบรรยากาศ  
- **LSWI (Land Surface Water Index)** ซึ่งใช้วิเคราะห์ความชื้นในพืชได้ดีในบางกรณี

### 3. ถ้าต้องนำผลลัพธ์นี้ไปใช้จริงกับหน่วยงานรัฐ อะไรคือข้อมูลที่ยังขาดอยู่?

แม้ว่าการวิเคราะห์นี้จะให้ภาพรวมของสภาพพืชพรรณและความชื้น แต่ยังขาดข้อมูลสำคัญ เช่น  
- ข้อมูลปริมาณน้ำฝน (rainfall data)  
- ข้อมูลการใช้ที่ดิน (land use/land cover)  
- ข้อมูลภาคสนาม (ground truth)  
- ข้อมูลแหล่งน้ำ เช่น อ่างเก็บน้ำหรือระบบชลประทาน  

การมีข้อมูลเหล่านี้จะช่วยเพิ่มความแม่นยำและทำให้สามารถนำผลลัพธ์ไปใช้ในการวางแผนจัดการทรัพยากรน้ำและการเกษตรได้จริง

### 4. การเลือก Scale ในการ Export มีผลต่อขนาดไฟล์และความแม่นยำอย่างไร?

การเลือก scale มีผลโดยตรงต่อทั้งขนาดไฟล์และรายละเอียดของข้อมูล  

- หากเลือก scale สูง (เช่น 30 เมตร หรือละเอียดมากขึ้น) จะได้ข้อมูลที่มีรายละเอียดสูง แต่ไฟล์จะมีขนาดใหญ่และใช้เวลาในการประมวลผลมาก  
- หากเลือก scale ต่ำ (เช่น 100 เมตร) จะช่วยลดขนาดไฟล์และเวลาในการประมวลผล แต่จะสูญเสียรายละเอียดเชิงพื้นที่  

ดังนั้น การเลือก scale จึงเป็นการ trade-off ระหว่าง “ความแม่นยำ” และ “ประสิทธิภาพในการจัดการข้อมูล” โดยในการศึกษานี้เลือกใช้ 30 เมตร เพื่อให้สอดคล้องกับความละเอียดของ Landsat 8 และรักษาคุณภาพของข้อมูลต้นฉบับ
