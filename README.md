## 💭 Discussion Questions

### ▸ 1. ถ้าเลือกดาวเทียมอีกดวง ผลการวิเคราะห์จะต่างออกไปอย่างไร? ลองทดสอบและเปรียบเทียบ

หากเลือกใช้ดาวเทียม Sentinel-2 ซึ่งมีความละเอียดเชิงพื้นที่สูงกว่า (10 เมตร) ผลการวิเคราะห์จะสามารถแสดงรายละเอียดของพื้นที่ได้ชัดเจนมากขึ้น โดยเฉพาะในระดับแปลงเกษตรหรือพื้นที่เมืองขนาดเล็ก อย่างไรก็ตาม ข้อมูลจะมีขนาดใหญ่ขึ้นและต้องใช้เวลาในการประมวลผลมากขึ้น

ในทางกลับกัน หากใช้ Landsat 9 ซึ่งมีคุณสมบัติใกล้เคียงกับ Landsat 8 ผลลัพธ์ที่ได้อาจไม่แตกต่างกันมากนัก แต่ความครอบคลุมของข้อมูลในบางช่วงเวลาอาจน้อยกว่า

ดังนั้น การเลือกดาวเทียมจึงเป็นการตัดสินใจเชิง trade-off ระหว่าง “ความละเอียดของข้อมูล” และ “ประสิทธิภาพในการประมวลผล”

### ▸ 2. Index ที่เลือกมีข้อจำกัดอะไร? มี Index อื่นที่เหมาะกว่าสำหรับพื้นที่นี้หรือไม่?

NDVI มีข้อจำกัดในการแยกชนิดพืช และอาจได้รับอิทธิพลจากพื้นผิวดิน (soil background)

NDWI ที่ใช้ในงานนี้เน้นการตรวจจับน้ำผิวดิน จึงไม่สามารถสะท้อนความชื้นในพืชได้โดยตรง ซึ่งสอดคล้องกับผลการวิเคราะห์ที่พบความสัมพันธ์เชิงลบกับ NDVI

NDMI แม้จะสะท้อนความชื้นในพืชได้ดีขึ้น แต่ยังมีข้อจำกัดในพื้นที่ที่มีความหลากหลายของการใช้ที่ดิน

Index อื่นที่อาจเหมาะสม ได้แก่

* **EVI (Enhanced Vegetation Index)** → ลดผลกระทบจากดินและบรรยากาศ
* **LSWI (Land Surface Water Index)** → วิเคราะห์ความชื้นในพืชได้ดีในบางบริบท

### ▸ 3. ถ้าต้องนำผลลัพธ์นี้ไปใช้จริงกับหน่วยงานรัฐ อะไรคือข้อมูลที่ยังขาดอยู่?

แม้ว่าผลการวิเคราะห์จะสามารถสะท้อนภาพรวมของสภาพพืชพรรณและความชื้นได้ดี แต่ยังขาดข้อมูลสำคัญ เช่น

* ข้อมูลปริมาณน้ำฝน (rainfall data)
* ข้อมูลการใช้ที่ดิน (land use/land cover)
* ข้อมูลภาคสนาม (ground truth)
* ข้อมูลแหล่งน้ำ เช่น อ่างเก็บน้ำหรือระบบชลประทาน

การบูรณาการข้อมูลเหล่านี้ร่วมกับข้อมูลดาวเทียมจะช่วยเพิ่มความแม่นยำ และทำให้ผลลัพธ์สามารถนำไปใช้ในการวางแผนและตัดสินใจเชิงนโยบายได้อย่างมีประสิทธิภาพมากขึ้น

### ▸ 4. การเลือก Scale ในการ Export มีผลต่อขนาดไฟล์และความแม่นยำอย่างไร?

การเลือก scale มีผลโดยตรงต่อทั้งความละเอียดของข้อมูลและขนาดไฟล์

* การใช้ scale ที่ละเอียด (เช่น 30 เมตร) จะให้ข้อมูลที่มีรายละเอียดสูง แต่ไฟล์มีขนาดใหญ่และใช้เวลาในการประมวลผลมาก
* การใช้ scale ที่หยาบลง จะช่วยลดขนาดไฟล์และเพิ่มความรวดเร็วในการประมวลผล แต่จะสูญเสียรายละเอียดเชิงพื้นที่

ดังนั้น การเลือก scale จึงเป็น trade-off ระหว่าง “ความแม่นยำ” และ “ประสิทธิภาพในการจัดการข้อมูล” โดยในการศึกษานี้เลือกใช้ 30 เมตร เพื่อให้สอดคล้องกับข้อมูล Landsat 8 และรักษาคุณภาพของข้อมูลต้นฉบับ
